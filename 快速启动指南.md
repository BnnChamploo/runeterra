# 符文之地论坛 - 快速启动指南

## 🎯 重要：项目需要Node.js 18+

当前系统Node版本：14.21.3（版本过低）

## ✅ 解决方案：使用nvm管理Node版本（不影响其他项目）

### 第一步：安装/使用nvm

nvm已经安装到你的系统中。要使用它，需要：

**方法A：重新打开终端窗口**（推荐）
- 关闭当前终端，重新打开一个新的终端窗口
- nvm会自动加载

**方法B：手动加载nvm**
```bash
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
```

### 第二步：安装Node 18（只需一次）

```bash
nvm install 18
```

### 第三步：在项目中使用Node 18

进入项目目录后，运行：
```bash
cd 项目根目录
nvm use 18
```

或者使用自动启动脚本：
```bash
./start.sh
```

## 📝 工作原理

1. **`.nvmrc`文件**：项目根目录有这个文件，指定使用Node 18
2. **nvm use**：会自动读取`.nvmrc`并切换到对应版本
3. **不影响其他项目**：
   - 系统默认Node版本（14.21.3）保持不变
   - 只有在这个项目目录中使用`nvm use 18`时才会切换
   - 其他项目继续使用系统默认版本

## 🚀 启动项目

### 方式一：使用启动脚本（最简单）

```bash
cd 项目根目录
./start.sh
```

### 方式二：手动启动

**终端1 - 启动后端：**
```bash
cd 项目根目录
nvm use 18  # 切换到Node 18
cd server
node index.js
```

**终端2 - 启动前端：**
```bash
cd 项目根目录
nvm use 18  # 切换到Node 18
cd client
npm run dev
```

## ❓ 常见问题

**Q: 会影响我其他项目吗？**
A: 不会！nvm只在你明确使用`nvm use`时切换版本。其他项目继续使用系统默认的Node 14。

**Q: 每次都要运行`nvm use 18`吗？**
A: 是的，但你可以：
- 使用启动脚本（自动处理）
- 或者配置shell自动切换（可选）

**Q: 如何查看当前使用的Node版本？**
A: 运行 `node -v` 查看

**Q: 如何切换回系统默认版本？**
A: 运行 `nvm use system`
